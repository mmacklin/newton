What Changed

Public API

  • JointType.ELASTIC owns a floating rigid frame plus reduced modal coordinates.
  • ModelBuilder.add_link_elastic() creates an elastic link for articulated mechanisms.
  • ModelBuilder.add_body_elastic() mirrors add_body() for stand-alone floating elastic bodies.
  • ModelBuilder.add_joint_elastic() is the state-owner joint used by those helpers.
  • ModalBasis stores sampled linear mode values; ModalGeneratorBeam, ModalGeneratorPOD, ModalGeneratorFEM, and ModalGeneratorSampled construct reusable bases.
  • The viewer now deforms elastic shape meshes from sampled modal basis values during log_state(), with an optional Matlab jet displacement-color diagnostic.

State Layout

  • joint_q: [px, py, pz, qx, qy, qz, qw, eta0, eta1, ...].
  • joint_qd: [vx, vy, vz, wx, wy, wz, eta_dot0, eta_dot1, ...].
  • The body still has a normal body_q and body_qd slot for rigid-body infrastructure.
  • Modal endpoint samples are cached from the existing joint parent/child transforms, with no separate attachment API.
  • Multiple elastic bodies can share one modal basis through elastic_basis; each joint/render sample keeps its basis-local sample index.

Solver Pipeline

Stage Implementation
Frame sync The elastic owner joint's first 7/6 coordinates are copied into the normal body pose/twist slots before rigid VBD runs.
Modal prediction Each modal coordinate uses an implicit Euler update with modal mass, stiffness, damping, and Control.joint_f.
Attachment evaluation Rigid VBD evaluates ordinary joint anchors through sampled mode shapes when the parent or child body is elastic.
Modal constraint solve After each rigid VBD pass, modal coordinates receive a diagonal implicit solve from joint-anchor residual gradients projected through the mode samples.
Frame write-back The solved body pose/twist is copied back into the owner joint, preserving the solved modal coordinates.
Viewer draw Builder finalization samples elastic shape vertices and a diagnostic centerline. ViewerGL hides the rigid instance for those shapes and uploads deformed mesh vertices each frame.

Mode Basis

The demo modes are now generated through ModalBasis rather than per-body Python callables. ModalBasis stores GPU-friendly samples phi[sample, mode]; builders add attachment and surface sample points locally to that basis, then flatten the values into the current VBD arrays.

ModalGeneratorFEM covers the Simscape-style matrix ROM path: users provide nodal coordinates plus M, K, and optional C matrices, Newton solves K phi = lambda M phi, mass-normalizes the selected eigenvectors, projects damping into modal space, and samples the resulting linear mode shapes at render vertices and joint interface nodes.

The beam generator produces standard small-deflection fields. The four-bar coupler uses one axial extension mode and the first two pinned-pinned Euler-Bernoulli bending modes. The bending displacement includes the plane-section axial term u_x = -y dw/dx, so the rendered beam cross-section rotates consistently with the neutral-axis deflection. The elastic box render cache now subdivides the visible surface in x, y, and z at roughly 1 cm spacing, and the beam examples seed each modal basis with those exact render-surface vertices, diagnostic line points, and attachment points.

The revolute endpoint fixture now uses eight independent linear POD modes extracted from finite-rotation torsion exemplar shapes. The initial 90 degree twist is a least-squares projection into that basis, so the held shape is nearly volume preserving without any example-side q_twist^2 coordinate coupling. After release, all modal coordinates evolve through the solver as ordinary linear modal DOFs. The example test tracks deformed mesh volume and fails if the visible shaft gains or loses too much volume.

The prismatic compression fixture uses an axial strain field with Poisson lateral strain. Its modal coordinate is driven from the same command as the prismatic actuator, so the doubled carriage stroke produces visible lateral bulging in compression and necking in extension while the fixed endpoints keep the elastic body in the joint hierarchy.

The mechanism set now includes three mixed rigid/elastic textbook-style articulations. A slider-crank uses the analytic closure equation for a connecting rod with elastic axial extension. A Watt straight-line linkage, historically used to approximate straight-line guided motion from rotary links, attaches both endpoints and a guided midpoint follower to one elastic coupler. The Watt drive is only the ground-to-left-rocker revolute target; the elastic modal DOFs are passive. A bell-crank transfer, a standard lever mechanism for redirecting motion through an angle, uses two separate elastic pushrods between rigid rockers; only the input rocker joint is actuated, and the elastic modes respond through the joint constraints. The Watt target is now tripled and the bell-crank target doubled, with longer clips to make the coupled rigid/elastic motion easier to read.

The gravity examples attach rigid weights to elastic bodies with ordinary fixed joints in a gravity-enabled model. A horizontal cantilever starts from an undeformed horizontal pose, drops under the attached weight, then runs with zero post-release modal damping while approaching the Euler-Bernoulli static tip displacement delta = P L^3 / (3 E I). A vertical bar is fixed at a raised top fixture, carries a heavier lower suspended weight, starts unextended, and checks the measured end-to-end stretch delta = |x_bottom - x_top| - L = P / k.

The flexible dipper example is currently set up as a static horizontal sag diagnostic. The elastic arm starts perfectly straight, is pinned at an interior fulcrum roughly two-thirds up a shortened vertical post, and is held by a fixed-length lower hydraulic link with zero actuator stroke. This isolates the gravity-loaded response of the reduced beam. The modal solve now projects the same translational joint damping force used by the rigid VBD joint evaluator, so the arm reaches its gravity sag quickly and settles instead of creeping slowly.

The vibration example uses a stiffer, lightly damped finite-mass mode and releases the beam from an initial deflection, producing a quick underdamped ring-down. ModalGeneratorPOD starts the exemplar-deformation path by extracting linear modes from sampled displacement snapshots.

Displacement Coloring

ViewerBase.show_elastic_strain is available as an optional diagnostic that colors each elastic render vertex by modal displacement magnitude relative to the rest shape. This is not true mechanical strain. The matrix ROM bracket and flexible dipper arm enable it to highlight where the modal response concentrates; the other report videos leave the diagnostic disabled and use the default orange elastic mesh color so the examples show the deformed geometry directly.

Revolute Torsion Fixture

Prismatic Compression Fixture

Elastic Slider-Crank

A rigid crank drives a rigid slider through an elastic connecting rod. The test compares slider position to the analytic slider-crank closure using the rod's current axial modal coordinate.

Elastic Watt Linkage

A Watt-style straight-line linkage constrains the elastic coupler endpoints through rigid rockers and attaches a guided midpoint follower to the same elastic body. Driven DOF: ground-to-left-rocker revolute target with tripled displacement. Elastic modal DOFs are passive.

Elastic Bell-Crank Transfer

Two elastic pushrods transmit motion from an actuated input rocker through a rigid bell crank to an output rocker, redirecting the input rotation through the central lever. The input rocker displacement is doubled, the elastic modes are not directly driven, and each pushrod is checked against its analytic modal length.

Matrix ROM Bracket

A perforated bracket starts from nodal mass, stiffness, and damping matrices. ModalGeneratorFEM extracts six mass-normalized modes; a low-damping kinematic gripper drives the fixed left interface through fast vertical move-and-hold segments, a rigid payload hangs from the right interface, and displacement coloring highlights the transient matrix-derived modal response.

Flexible Dipper Arm

A straight-start dipper arm is pinned at an interior fulcrum on a shortened support post, held by a fixed-length lower hydraulic link, and loaded at the tool end. With piston actuation disabled, the clip shows the gravity sag settling response after the modal solve includes the same translational joint damping force as the rigid joint path.

Cantilever Tip Load

A single elastic link uses the exact Euler-Bernoulli cantilever tip-load shape. The modal coordinate tracks the analytic static solution as the load varies.

Cantilever Vibration

The same reduced beam formulation is given finite modal mass and released from an initial deflection, so the visible motion comes from modal dynamics rather than a prescribed load.

Cantilever With Rigid Weight

A fixed elastic beam starts horizontal, carries a rigid tip mass in gravity, and uses zero post-release modal damping. The reduced coordinate uses the exact Euler-Bernoulli tip-load shape, and the final deflection is checked against P L^3 / (3 E I).

Vertical Suspended Weight

A vertical elastic bar is fixed at a raised top fixture and supports a much heavier rigid block under gravity. It starts unextended, visibly stretches, and the top-fixed axial mode includes Poisson lateral contraction while the example checks measured total stretch rather than compression.

Current Scope

  • Endpoint, solver, and viewer modes are translational FFRF samples. Mesh rendering now evaluates the same basis at each render vertex.
  • The modal constraint solve is diagonal per mode. This keeps the implementation small and robust for few-mode links, but dense modal coupling is still future work.
  • Physical modal solving is implemented for VBD. Other solvers can carry the state layout but do not yet provide the same elastic-mode constraint solve.
  • Shape deformation is currently a viewer path; collision shapes are still handled by the rigid-body collision pipeline.
  • Existing parent_xform and child_xform arguments remain the attachment API for now.
