What Changed

Public API

  • JointType.ELASTIC owns a floating rigid frame plus reduced modal coordinates.
  • ModelBuilder.add_link_elastic() creates an elastic link for articulated mechanisms.
  • ModelBuilder.add_body_elastic() mirrors add_body() for stand-alone floating elastic bodies.
  • ModelBuilder.add_joint_elastic() is the state-owner joint used by those helpers.
  • The viewer now renders reduced elastic body centerlines from sampled modal shape data during log_state().

State Layout

  • joint_q: [px, py, pz, qx, qy, qz, qw, eta0, eta1, ...].
  • joint_qd: [vx, vy, vz, wx, wy, wz, eta_dot0, eta_dot1, ...].
  • The body still has a normal body_q and body_qd slot for rigid-body infrastructure.
  • Modal endpoint samples are cached from the existing joint parent/child transforms, with no separate attachment API.

Solver Pipeline

Stage Implementation
Frame sync The elastic owner joint's first 7/6 coordinates are copied into the normal body pose/twist slots before rigid VBD runs.
Modal prediction Each modal coordinate uses an implicit Euler update with modal mass, stiffness, damping, and Control.joint_f.
Attachment evaluation Rigid VBD evaluates ordinary joint anchors through sampled mode shapes when the parent or child body is elastic.
Modal constraint solve After each rigid VBD pass, modal coordinates receive a diagonal implicit solve from joint-anchor residual gradients projected through the mode samples.
Frame write-back The solved body pose/twist is copied back into the owner joint, preserving the solved modal coordinates.
Viewer draw Builder finalization samples each elastic body's mode shapes along a local render polyline. ViewerGL draws the deformed centerline and endpoint markers automatically.

Current Scope

  • Endpoint and viewer modes are translational samples. The example uses one axial mode plus quadratic and sinusoidal bending modes with zero endpoint displacement.
  • The modal constraint solve is diagonal per mode. This keeps the implementation small and robust for few-mode links, but dense modal coupling is still future work.
  • Physical modal solving is implemented for VBD. Other solvers can carry the state layout but do not yet provide the same elastic-mode constraint solve.
  • Existing parent_xform and child_xform arguments remain the attachment API for now.
